# Changelog

## [Unreleased]

### Added

- Merge non-fetch handlers (`scheduled`, `queue`, `email`, `tail`, `trace`)
  from the entry's `default` export onto the production worker's default
  export. Previously these were silently dropped because Cloudflare invokes
  them as methods on the default object, not as named exports.
  ([#3](https://github.com/oselvar/sveltekit-add-worker-exports/issues/3))
- Enabled `testScheduled` on the dev sidecar so `scheduled` handlers can be
  invoked via `curl http://localhost:8787/__scheduled?cron=...` — wrangler
  dev never auto-fires crons.

### Fixed

- Honor `CLOUDFLARE_ENV` when rewriting `script_name` in the platform-proxy
  config. Wrangler suffixes the sidecar's registered name with the env name,
  so cross-worker bindings now reference the suffixed name and resolve via
  the dev registry. The plugin also patches the active `env[envName]` block
  instead of the top-level, since per-env `durable_objects` / `workflows`
  arrays wholly override the top-level rather than merging.
  ([#2](https://github.com/oselvar/sveltekit-add-worker-exports/issues/2))

### Changed

- Consolidated the example's WebSocket `fetch` and new `scheduled` handler
  into a single `default` export in `src/lib/server/index.ts`. Removed the
  `devHandler.ts` indirection, since `scheduled` makes "dev" a misnomer.

## [0.3.2](https://github.com/oselvar/sveltekit-add-worker-exports/compare/v0.3.1...v0.3.2) (2026-06-04)

### Changed

- Upgraded `wrangler` peer/dev dependency floor to 4.98.0.
- Pointed the dev sidecar's `main` directly at the user entry point now that
  `getPlatformProxy` preserves workflow bindings with `script_name`
  ([cloudflare/workers-sdk#13863](https://github.com/cloudflare/workers-sdk/pull/13863)).
- Reduced `@oselvar/sveltekit-add-worker-exports/hooks` to a no-op handle;
  the export is kept for backwards compatibility but no longer needs to be
  wired into `hooks.server.ts`.

### Removed

- Removed the `./dev-bridge` package export and the internal `src/bridge/`
  HTTP fallback that synthesized Workflow bindings under unpatched
  wrangler/miniflare.

### Tooling

- Migrated release tooling from `np` to `release-it` with the
  `@release-it/conventional-changelog` plugin. Future releases follow
  Conventional Commits.
